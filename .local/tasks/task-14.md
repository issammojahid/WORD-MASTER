## الهدف
حجب الاشتراك المجاني لـ VIP وحزم العملات المجانية، وتحديث الأسعار بشكل صحيح.

## المشكلات الحالية
1. **app/vip.tsx** — `handleSubscribe()` كيكل API مباشرةً (`/activate-vip`) بدون أي دفع حقيقي → VIP مجاني
   - السعر مكتوب "100 درهم / شهر" — غلط
2. **app/shop.tsx** — `handleCoinPack()` كتستنا 1200ms وكتضيف العملات مباشرةً بالمجان:
   - 500 عملة مجانية ("مجاناً مع الإعلان")
   - 1200+200 عملة مجانية ("عرض خاص")
   - 3000+800 عملة مجانية ("قيمة مضاعفة")
   - لا يوجد أي payment check أو API call حقيقي

## التعديلات المطلوبة

### 1. app/vip.tsx
- **غير السعر**: "100 درهم / شهر" → **"1.99€ / شهر"** مع subtitle "اشتراك شهري متجدد"
- **احجب زر الاشتراك**: عوض ما تكل API، بين modal جميل "نظام الدفع قيد الإعداد":
  - Emoji 🔧 كبير
  - عنوان: "سيتم إطلاق نظام الدفع قريباً"
  - نص: "نحن نعمل على دمج خدمة الدفع. شكراً على صبرك 🙏"
  - زر "حسناً" لإغلاق الـ modal
  - نفس الـ dark gold color palette ديال الشاشة (background #0A0A1A، border ذهبي، نص فاتح)
- ما تمسحش أي شي من الواجهة (الفوائد، التصميم، التأنيمات) — فقط الزر يعوض الدفع بالـ modal

### 2. app/shop.tsx — حزم العملات
- **احجب `handleCoinPack()`**: عوض ما تعطي عملات مجانية، بين نفس modal "نظام الدفع قيد الإعداد":
  - نفس التصميم (modal صغير، dark، gold border)
  - نص: "سيتم دعم الدفع الإلكتروني قريباً"
  - زر "حسناً" لإغلاق الـ modal
- **غير labels الأسعار** في `COIN_PACKS`:
  - starter (500): "0.99€"
  - medium (1200+200): "2.99€"
  - premium (3000+800): "5.99€"
- ما تمسحش الـ UI cards، الـ gradients، أو الـ badges

## ما لا يتغير
- Mystery boxes (كتستعمل عملات داخلية — صح)
- Battle Pass premium (كيتبع عملات داخلية — صح)
- Daily shop items (كتستعمل عملات داخلية — صح)
- Spin wheel, avatars, effects, titles — كلها بعملات داخلية — صح
- كل الـ navigation, animations, socket connections

## النتيجة المتوقعة
- VIP يبين "1.99€ / شهر" ولما تضغط "اشترك" تبين رسالة "نظام الدفع قيد الإعداد 🔧"
- حزم العملات تبين أسعار حقيقية بالأورو ولما تضغط "احصل عليه" تبين نفس الرسالة
- باقي المتجر يشتغل عادي بالعملات الداخلية
